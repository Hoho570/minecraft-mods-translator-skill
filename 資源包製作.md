# Minecraft 模組翻譯資源包製作指南

這份指南將教你如何創建一個標準的 Minecraft 資源包（Resource Pack），專門用來覆蓋或新增模組（Mod）的語言翻譯。

---

## 1. 準備工作

在開始之前，建議準備以下工具：
- **文字編輯器**：推薦使用 [VS Code](https://code.visualstudio.com/) 或 [Notepad++](https://notepad-plus-plus.org/)。
  - *注意：請勿使用 Windows 內建記事本，以免發生 UTF-8 編碼錯誤。*
- **解壓縮軟體**：如 [7-Zip](https://www.7-zip.org/) 或 [WinRAR](https://www.rarlab.com/)。

---

## 2. 建立資源包結構

資源包必須遵循特定的資料夾結構。請建立如下目錄：

```text
我的翻譯包/
├── pack.mcmeta          (必要：設定檔)
├── pack.png             (選用：資源包圖示)
└── assets/
    └── <模組ID>/        (例如：ironchest)
        └── lang/
            └── zh_tw.json
```

### 關鍵路徑說明
- **`<模組ID>` (Namespace)**：這是最關鍵的步驟。
  - **如何查找？** 用解壓縮軟體打開模組的 `.jar` 檔，進入 `assets` 資料夾，其中的資料夾名稱即為 ID。
  - *範例：`jei-1.19.2.jar` 內的 `assets/jei`，其 ID 即為 `jei`。*
- **`lang`**：存放語言檔的固定資料夾名稱。
- **`zh_tw.json`**：繁體中文（台灣）語言檔。
  - *1.12.2 或更舊版本通常為 `zh_TW.lang`。*

---

## 3. 設定 pack.mcmeta

在根目錄建立 `pack.mcmeta`，並填入以下內容：

```json
{
  "pack": {
    "pack_format": 15,
    "description": "我的模組翻譯資源包"
  }
}
```

### pack_format 版本對照表

| Minecraft 版本 | pack_format |
| :--- | :--- |
| 1.21 | 34 |
| 1.20.6 | 32 |
| 1.20.4 | 22 |
| 1.20.2 | 18 |
| 1.20.1 | 15 |
| 1.19.4 | 13 |
| 1.19.3 | 12 |
| 1.19 | 9 |
| 1.18.2 | 8 |
| 1.16.5 | 6 |
| 1.12.2 | 3 |

## 4. 提取與翻譯流程

### 第一步：取得原始檔
1. 使用 7-Zip 打開模組 `.jar`。
2. 進入 `assets/<模組ID>/lang/`。
3. 複製 `en_us.json` (或 `en_us.lang`)。

### 第二步：建立中文檔
1. 將檔案重新命名為 `zh_tw.json`。
   - **版本差異**：1.13+ 須全小寫 (`zh_tw`)；1.12.2 以前通常包含大寫 (`zh_TW`)。
2. 放入資源包對應路徑：`assets/<模組ID>/lang/zh_tw.json`。

### 第三步：進行翻譯
編輯 `zh_tw.json` 內容：

```json
{
  "item.ironchest.iron_chest": "鐵箱子",
  "block.ironchest.gold_chest": "黃金箱子",
  "message.ironchest.locked": "箱子已上鎖！"
}
```

- **左側 (Key)**：程式鍵值，**絕對不能修改**。
- **右側 (Value)**：遊戲顯示文字，請在此翻譯。
- **語法注意**：
  - 每行結尾需有逗號 `,`（最後一行除外）。
  - 使用半形引號 `"`。
  - 特殊符號需轉義，例如：`\"`。
  - 保留變數符號（如 `%s`, `%1$s`）。

## 常見問題 (FAQ)

- **Q: 為什麼翻譯沒顯示？**
  - 檢查資料夾結構是否為 `assets/<模組ID>/lang/`。
  - 檢查 JSON 語法（建議使用 [JSON Validator](https://jsonlint.com/)）。
  - 確認遊戲語言已切換為「繁體中文（台灣）」。
- **Q: 字體顯示為方塊？**
  - 可能該字型不支援特定中文字，建議安裝支援 Unicode 的字體模組。
- **Q: 1.12.2 舊版格式？**
  - 使用 `.lang` 格式，結構為 `key=value`（不需引號），例如：`tile.ironchest.name=鐵箱子`。
